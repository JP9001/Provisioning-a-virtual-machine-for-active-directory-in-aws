# Creating a Virtual Machine for Active Directory on AWS

## Objective

This Standard Operating Procedure (SOP) outlines the steps to create a Windows virtual machine for hosting Active Directory using **Amazon EC2** in the **AWS Management Console**. Following these steps ensures a smooth and efficient deployment process.

---

## Prerequisites

- Active AWS account
- Permissions to create EC2 instances
- Existing key pair (or ability to create one)
- Basic understanding of Remote Desktop Protocol (RDP)

---

## Step 1: Launch an EC2 Instance

1. Log in to the **AWS Management Console**.
2. Navigate to **EC2 Dashboard**.
3. Click **Launch Instance**.

---

## Step 2: Select Windows (Quick Start AMI)

1. In the **Application and OS Images (Amazon Machine Image)** section:
   - Select **Windows** under **Quick Start**.
2. Choose the appropriate Windows Server version (e.g., Windows Server 2019 or 2022).
